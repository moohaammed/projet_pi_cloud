# Smartwatch Heart-Rate Service

A Spring Boot microservice that collects, stores, and serves real-time heart-rate data from BLE smartwatches.  
Part of the **MediSync** microservices architecture.

---

## 📋 Overview

| Property           | Value                              |
| ------------------ | ---------------------------------- |
| **Service name**   | `smartwatch-service`               |
| **Port**           | `8095`                             |
| **Database**       | MongoDB — `smartwatch_db`          |
| **Package**        | `tn.esprit.smartwatchservice`      |
| **Spring Boot**    | 3.3.0                              |
| **Java**           | 17                                 |
| **Spring Cloud**   | 2023.0.2                           |

## 🔌 REST API Endpoints

Base path: `/api/heart-rate`

| Method   | Path                          | Description                          | Response         |
| -------- | ----------------------------- | ------------------------------------ | ---------------- |
| `POST`   | `/api/heart-rate`             | Save a new heart-rate reading        | `201 Created`    |
| `GET`    | `/api/heart-rate/latest/{userId}` | Get the most recent reading      | `200` or `204`   |
| `GET`    | `/api/heart-rate/history/{userId}` | Get all readings for a user     | `200` or `204`   |
| `GET`    | `/api/heart-rate/{id}`        | Get a specific record by ID          | `200` or `404`   |
| `DELETE` | `/api/heart-rate/{id}`        | Delete a specific record             | `204 No Content` |

### Request Body (POST)

```json
{
  "userId": 1,
  "deviceName": "Mi Band 7",
  "bpm": 78
}
```

> **Note:** The `recordedAt` field is automatically generated by the server when the heart-rate record is saved. Clients must not send this field.

## 🚀 Running the Service

### Prerequisites
- Java 17
- MongoDB running on `localhost:27017`
- Eureka Server running on `localhost:8761`
- API Gateway running on `localhost:8080`

### Start the service

```bash
cd smartwatch-service
mvnw.cmd spring-boot:run    # Windows
./mvnw spring-boot:run       # Linux/Mac
```

### Verify registration

1. Open the Eureka Dashboard: http://localhost:8761
2. Confirm `SMARTWATCH-SERVICE` appears in the registered instances list.

### Test through the API Gateway

```bash
# Save a reading
curl -X POST http://localhost:8080/api/heart-rate \
  -H "Content-Type: application/json" \
  -d '{"userId":1,"deviceName":"Test","bpm":72}'

# Get latest
curl http://localhost:8080/api/heart-rate/latest/1

# Get history
curl http://localhost:8080/api/heart-rate/history/1
```

## 🌐 API Gateway Integration

The following route is configured in `api-gateway/src/main/resources/application.yml`:

```yaml
- id: smartwatch-service
  uri: lb://smartwatch-service
  predicates:
    - Path=/api/heart-rate/**
```

> This route is placed **before** the catch-all `backpi` route to ensure correct routing.

## 🖥️ Angular Frontend Integration

### Components

| File | Purpose |
| ---- | ------- |
| `services/heart-rate.service.ts` | HTTP service to communicate with the API |
| `components/live-heart-rate/` | Dashboard component with live ECG visualization |

### Route

```ts
{ path: 'heart-rate', loadComponent: () =>
  import('./components/live-heart-rate/live-heart-rate.component')
  .then(m => m.LiveHeartRateComponent)
}
```

### Navigation

A "Heart Rate" link with `fa-heart-pulse` icon is added to the standard layout navbar.

### Features

- ❤️ Real-time BPM display with color-coded zones
- 📈 Live ECG-style canvas waveform with glow effects
- 📊 Statistics: average, min, max BPM
- 📋 History table with zone indicators
- ⚠️ Disconnection detection (6-second threshold)
- 🔄 Auto-polling every 2 seconds

## 🐍 Python BLE Client

Located in `scripts/smartwatch-client/`.

### Install dependencies

```bash
pip install bleak requests
```

### Run with a real smartwatch

```bash
python smartwatch_ble_client.py
```

### Run in simulation mode (no smartwatch needed)

```bash
python smartwatch_ble_client.py --simulate
```

This sends realistic random BPM values every 2 seconds to test the full stack.

### Configuration (in the script)

```python
DEVICE_NAME = "Mi Band"          # BLE device name to search for
API_URL = "http://localhost:8080/api/heart-rate"
USER_ID = 1                      # User ID to associate readings with
```

## 🔐 Authentication Notes

- This service stores only `userId` (Long) — no direct joins with user tables.
- The Angular component attempts to read the user ID from `AuthService.getCurrentUser().id`.
- If no user is logged in, it defaults to `userId = 1`.
